# Branch and Commit Guidelines

## Branch Naming

- `feat/<short-topic>`
- `fix/<short-topic>`
- `refactor/<short-topic>`
- `docs/<short-topic>`
- `chore/<short-topic>`

## Commit Message Format

`<type>: <summary>`

Types:

- `feat`, `fix`, `refactor`, `docs`, `chore`, `test`

Examples:

- `fix: handle non-json trade API response safely`
- `feat: add readiness endpoint for operational checks`
