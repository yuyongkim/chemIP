.venv/                 ❌ Python 가상환경
node_modules/          ❌ npm 패키지
.env                   ❌ API 키 등 비밀 정보
*.db                   ❌ 데이터베이스 파일
*.log                  ❌ 로그 파일

🗄️ 140GB 데이터베이스 공유 방법

Git으로 올릴 수 없으므로, 아래 방법 중 선택:

방법 장점 단점
1. NAS/공유 드라이브 팀 내부에서 직접 접근 가능 외부 접근 불가
2. 클라우드 (S3, GCS, Azure) 어디서든 다운로드 가능 비용 발생
3. 외장하드 전달 가장 빠른 전송 물리적 전달 필요
4. 로컬에서 재생성 항상 최신 데이터 24-48시간 소요

권장: 클라우드 또는 NAS 공유

# 예시: AWS S3 업로드
aws s3 cp data/global_patent_index.db s3://your-bucket/msds-db/

# 다운로드
aws s3 cp s3://your-bucket/msds-db/global_patent_index.db data/

🖥️ 백엔드 운영 방법

개발 환경 (로컬)

# 1. 가상환경 활성화 + 서버 실행
cd C:\Users\USER\Desktop\MSDS
.\.venv\Scripts\activate
uvicorn backend.main:app --reload --port 8000

프로덕션 환경

Option 1: 서버에 직접 배포

# 1. 서버에 코드 복사
git clone [your-repo-url]
cd MSDS

# 2. 가상환경 설정
python -m venv .venv
source .venv/bin/activate  # Linux/Mac
pip install -r requirements.txt

# 3. 데이터베이스 다운로드 (S3 등에서)
aws s3 cp s3://bucket/global_patent_index.db data/

# 4. 프로덕션 서버 실행 (Gunicorn 권장)
pip install gunicorn
gunicorn backend.main:app -w 4 -k uvicorn.workers.UvicornWorker --bind 0.0.0.0:8000

Option 2: Docker로 배포 (권장)

# Dockerfile
FROM python:3.11-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY backend/ backend/
COPY data/ data/  # 또는 볼륨 마운트
EXPOSE 8000
CMD ["uvicorn", "backend.main:app", "--host", "0.0.0.0", "--port", "8000"]

docker build -t chemip-backend .
docker run -d -p 8000:8000 -v /path/to/data:/app/data chemip-backend

🔄 협업 워크플로우

새 팀원이 프로젝트 시작할 때

# 1. 코드 클론
git clone [your-repo-url]
cd MSDS

# 2. Python 가상환경 설정
python -m venv .venv
.\.venv\Scripts\activate
pip install -r requirements.txt

# 3. .env 파일 생성 (API 키 입력)
# KOSHA_SERVICE_KEY_DECODED=xxxxx
# KIPRIS_API_KEY=xxxxx

# 4. 데이터베이스 다운로드 (팀에서 공유받기)
# 방법 A: NAS에서 복사
xcopy \\NAS\shared\msds\data\* data\

# 방법 B: 클라우드에서 다운로드
aws s3 sync s3://bucket/msds-data/ data/

# 방법 C: 직접 생성 (24-48시간 소요)
python scripts/build_global_index.py

# 5. 프론트엔드 설정
cd frontend
npm install

# 6. 실행
cd ..
uvicorn backend.main:app --reload --port 8000
# 새 터미널에서
cd frontend && npm run dev

📋 요약 체크리스트

Git Push 전 확인

  • .gitignore에 data/, .env, *.db 포함 확인
  • 민감한 API 키가 코드에 하드코딩되어 있지 않은지 확인
  • requirements.txt 최신화 (pip freeze > requirements.txt)

팀원 온보딩 시

  • Git 저장소 접근 권한 부여
  • .env 파일 내용 공유 (슬랙 DM 등 보안 채널로)
  • 140GB DB 파일 공유 방법 안내
  • 원본 특허 데이터 경로 안내 (S:\특허 논문 DB\...)

백엔드 배포 시

  • data/global_patent_index.db 서버에 복사 완료
  • .env 환경변수 설정
  • 포트 8000 방화벽 오픈
  • SSL 인증서 설정 (HTTPS)
