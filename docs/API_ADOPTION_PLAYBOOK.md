# ChemIP Platform 활용계획서 (제출용)

## 1. 사업 개요
### 1.1 문서 목적
본 문서는 화학물질 관련 행정·지원 업무의 효율화를 위해 ChemIP Platform의 도입 필요성, 적용 범위, 운영 방안 및 검증 계획을 제시하기 위한 제출용 문서입니다.

### 1.2 한 줄 요약
**화학물질 검색 1회로 안전정보(MSDS), 특허, 시장/진출 정보를 통합 조회하여 초기 검토의 속도와 일관성을 개선하는 실무 지원 플랫폼**

---

## 2. 추진 배경 및 필요성
### 2.1 현행 업무의 한계
- 정보 소스 분산: 안전정보, 특허, 시장자료가 서로 다른 채널에 분산
- 조사 방식 비표준: 담당자별 조사 경로·문서 형식 상이
- 의사결정 지연: 초기 검토 자료 취합에 과도한 시간이 소요
- 리스크 관리 한계: 규제·특허 이슈의 사후 발견 가능성 존재

### 2.2 개선 방향
- 단일 화면 기반 통합 조회 체계 구축
- 부서 간 동일 기준 데이터 공유
- 파일럿 결과 검증 후 단계적 확산

### 2.3 수치 표기 원칙(제출용)
- 정량 수치는 파일럿 측정값이 확보된 경우에만 사용
- 측정값 제시 시 측정기간, 표본, 산식을 함께 기재
- 측정 전 단계에서는 \"유의미한 개선\", \"체감 단축\" 등 정성 표현 사용

---

## 3. 서비스 범위
### 3.1 제공 기능
1. 화학물질 통합 검색
2. MSDS 관련 정보 조회
3. 특허 정보 조회(국가/문헌 기반)
4. 시장/진출 참고 정보 조회

### 3.2 책임 경계(중요)
- 본 시스템은 **사전 스크리닝 및 의사결정 지원 도구**입니다.
- 법률 자문, 최종 규제 해석, 최종 특허 침해 판단은 담당 부서 및 전문가 검토 절차를 통해 확정합니다.

권장 표기 문구:
> "본 시스템은 사전 검토를 지원하며, 최종 판단은 관련 법령 및 전문 검토 절차에 따라 확정한다."

---

## 4. 적용 시나리오 (실무 중심)
### 4.1 국내 중소기업 해외 진출 사전검토 지원
- 목적: 수출 대상 물질의 초기 규제·특허 리스크 점검
- 기대: 1차 판단 리드타임 단축, 재검토 감소

### 4.2 해외기업 국내 진출 컴플라이언스 사전점검
- 목적: 국내 유통 전 요구사항(안전자료/규제/특허) 선별
- 기대: 문서 보완·재작업 감소, 일정 지연 완화

### 4.3 정책/사업부 초기 타당성 검토
- 목적: 기술성 외 규제·특허 리스크를 초기 단계에서 반영
- 기대: Go/No-Go 판단의 객관성 및 일관성 강화

## 5. 운영 경험 기반 실제 개선 사례 (익명화)
적용 기간: 2026년 2월 14일 ~ 2026년 2월 15일

| 항목 | 확인된 문제 | 조치 내용 | 확인 결과 |
|---|---|---|---|
| 시장/진출 조회 | 특정 키워드에서 원천 API 0건 반환 | 0건 시 대체 조회(fallback) 적용 | 빈 화면 대신 참고 데이터 제공 |
| 진출전략 화면 | 국가 나열 중심으로 판단 정보 부족 | 요약본/실행 과제/리스크 중심으로 UI 개선 | 보고·시연 설명력 개선 |
| 무역사기 데이터 | 본문 인코딩/가독성 이슈 | 본문 정규화 + 핵심 필드 추출 | 필터링 및 핵심 확인 용이 |
| 안전정보 수집 | 일부 물질 영어명 기반 매칭 한계 | GHS 안전문구 직접 수집 배치로 전환 | 대량 수집 파이프라인 확보 |

주의: 상기 사례는 운영 개선 이력이며, 대외 성과 수치로 사용 시 별도 KPI 검증이 필요합니다.

## 6. 시연 계획(5분)
### 6.1 시연 순서
1. 문제 정의(1분): 분산 조회 구조의 한계 설명
2. 라이브 조회(2분): 물질 검색 -> MSDS/특허/시장 탭 확인
3. 운영 방식(1분): API + 로컬 인덱스 하이브리드 및 fallback 설명
4. 확산 계획(1분): 파일럿 결과 검증 후 단계 확산

### 6.2 시연 시 강조 포인트
- "하나의 화면에서 초기 판단 자료를 표준화해 제공"
- "원천 API 공백 시에도 대체 경로로 업무 연속성 유지"

---

## 7. 보안 및 운영 관리 방안
### 7.1 보안 원칙
- 접근제어(예: Zero Trust)
- CORS 최소 허용
- API 키 분리 보관(.env)
- 로그 보존 및 감사 추적

### 7.2 장애 대응 원칙
- 핵심 기능은 로컬 데이터 기반으로 우선 유지
- 외부 연동 구간은 단계적 저하(degraded mode) 적용
- 장애 발생 시 기록/복구/재발방지 액션 관리

---

## 8. 도입 및 확산 계획(사례 중심)
### 8.1 즉시 적용(1주)
- 시연 대상 부서 선정(안전/특허/사업 중 1~2개)
- 대표 물질 10건을 선정해 실제 업무 흐름으로 시연
- 현행 방식 대비 \"자료 취합 절차\"가 어떻게 단축되는지 확인

### 8.2 파일럿 운영(2~4주)
- 실제 민원/검토 업무에 제한 적용
- 주 1회 운영 점검: 데이터 공백, fallback 동작, 사용자 불편사항
- 개선 요청을 즉시 반영하고 변경 이력을 기록

### 8.3 확산 의사결정(1개월+)
- 파일럿 결과를 부서장 회의에서 공유
- 확산 범위(대상 부서/사용자/업무 유형) 단계적 확대
- 운영 책임(관리부서, 장애 연락체계, 문서관리)을 확정

### 8.4 제출본에서 강조할 문장
- \"본 시스템은 현행 업무를 대체하는 것이 아니라, 초기 검토 품질과 속도를 보완하는 도구입니다.\"
- \"도입은 전면 전환이 아닌 파일럿-검증-확산의 단계적 방식으로 진행합니다.\"
- \"규제·특허 최종 판단은 기존 결재 및 전문가 검토 절차를 유지합니다.\"

## 9. QA 세션 대응 표준 답변
### Q1. 정확도 수치가 있는가?
A. 단일 정확도 수치 대신 항목별 품질지표(누락률, 재작업률, 정정건수)로 검증합니다.

### Q2. 외부 API 장애 시 서비스는?
A. 로컬 데이터 기반 기능을 우선 유지하고, 외부 연동 구간은 대체 경로를 적용합니다.

### Q3. 법적 판단 대체가 가능한가?
A. 불가합니다. 본 시스템은 사전 검토 도구이며 최종 판단은 별도 절차를 따릅니다.

### Q4. 비용 절감 근거는?
A. 비용·시간 절감 수치는 사전 추정이 아니라 파일럿 전후 비교 측정값으로 제시합니다. 측정 전 단계에서는 정성 효과만 설명합니다.

### Q5. 보안 통제는 어떻게 하나?
A. 접근제어, 키 분리보관, 최소 권한, 로그 관리 원칙을 적용합니다.
