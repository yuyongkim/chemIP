## 후속 확장 논문 3개 기획안

### 1. 실사용 검증 논문: ChemIP가 실제 업무 시간을 얼마나 줄이는가
**한 줄 주제**
- ChemIP 사용 전/후를 비교하여 화학물질 조사 업무의 시간, 누락률, 재작업률 개선을 검증하는 실증 논문.

**핵심 질문**
- ChemIP가 실제 조사 시간을 줄이는가?
- 정보 누락을 줄이는가?
- 초보자와 숙련자 간 품질 편차를 줄이는가?

**권장 평가 지표**
- 과제 완료 시간
- 필수 정보 누락률
- 재검색/재작업 횟수
- 사용자 만족도(간단 설문)

**실험 설계 초안**
- 시나리오 3~5개: 수출 사전검토, 국내 진입 검토, 의약품 교차참조 등
- 참가자 6~12명: 학생/연구원/실무자 혼합 가능
- 비교군: 기존 포털 직접 조회 vs ChemIP 사용

**왜 좋은가**
- 현재 제출 논문들의 가장 약한 부분인 "효과 검증"을 직접 보강함.
- ACS CHS 계열 후속 논문으로 특히 유리함.

**리스크**
- 사람을 모아야 하고 실험 설계가 필요함.
- IRB 수준까지는 아닐 수 있어도 윤리/동의 문구는 준비 필요.

### 2. 데이터 품질/정규화 논문: 한국 화학물질 검색 정규화와 엔터티 매칭
**한 줄 주제**
- 한국어/영어/동의어/CAS 번호가 섞인 화학물질 검색에서 entity normalization이 검색 성공률을 얼마나 높이는지 다루는 데이터 중심 논문.

**핵심 질문**
- 화학물질명 표기 변형 때문에 검색 실패가 얼마나 발생하는가?
- 정규화 사전과 규칙 기반 매칭이 검색 recall을 얼마나 올리는가?
- KOSHA/KISCHEM/NCIS/MFDS/OpenFDA/PubChem 사이의 명칭 불일치를 얼마나 줄일 수 있는가?

**권장 기여 포인트**
- 한국어-영어-약칭-별칭-CAS 매핑 사전 구축
- 규칙 기반 + 사전 기반 정규화 파이프라인
- 검색 성공률/정확도/empty-result 감소율 평가

**데이터셋 아이디어**
- 자주 조회하는 물질 300~1000개 curated set
- 동의어/오탈자/영문표기 변형 포함
- 기관별 상이한 명칭 케이스 수집

**왜 좋은가**
- 제품 개선과 논문 기여가 직접 연결됨.
- 후속 기능(검색 품질, 추천 정확도)과도 맞물림.
- Journal of Cheminformatics 류와 궁합이 좋음.

**리스크**
- 데이터 정제 노동이 큼.
- gold standard를 만드는 기준을 명확히 해야 함.

### 3. 증거 기반 안전요약 논문: LLM보다 deterministic synthesis 중심
**한 줄 주제**
- 여러 규제/안전/특허 소스에서 수집한 근거를 구조화하여, 신뢰 가능한 안전요약을 생성하는 evidence synthesis 엔진에 대한 논문.

**핵심 질문**
- LLM 없이도 충분히 유용한 안전요약을 만들 수 있는가?
- source attribution과 confidence scoring이 사용자 신뢰를 높이는가?
- 규칙 기반 요약과 LLM 보조 요약의 역할을 어떻게 분리할 수 있는가?

**권장 기여 포인트**
- evidence bundle schema 정의
- 출처별 우선순위/충돌 해결 규칙
- confidence score 계산 방식
- fallback-first 설계: 기본은 deterministic, 선택적으로 LLM 보조

**평가 방법**
- 전문가 혹은 준전문가가 읽고
  - 근거 추적 가능성
  - 요약의 완결성
  - 위험 정보 누락 여부
  - 신뢰도
  를 평가

**왜 좋은가**
- 현재 논문에서 민감할 수 있는 LLM 이슈를 오히려 강점으로 전환 가능.
- 안전 분야에서 "설명 가능성"과 "근거 기반"을 강조할 수 있음.

**리스크**
- 좋은 schema와 평가 rubric이 필요함.
- 규칙 기반이면 참신성이 약해 보일 수 있어 설계 철학을 잘 보여줘야 함.

5. **ChemIP 사용자 연구: 화학물질 사전검토 워크플로우에서 통합 조회 시스템의 효과**

#### 2) 초록 초안
화학물질의 규제·안전성 검토 업무는 일반적으로 물질안전보건자료(MSDS), 특허, 시장·진출 정보, 의약품 관련 데이터 등 여러 분절된 포털을 순차적으로 조회하는 방식으로 수행된다. 이러한 방식은 조사 시간 증가, 정보 누락, 담당자별 결과 편차를 초래한다. 본 연구는 한국 및 국제 화학안전 정보를 통합 조회하는 오픈소스 플랫폼 ChemIP가 실제 업무 수행에 미치는 영향을 과제 기반 사용자 평가를 통해 검증한다. 참가자들은 기존 포털 기반 방식과 ChemIP 기반 방식을 각각 사용하여 수출 사전검토, 국내 진입 검토, 의약품 교차참조 등 대표 업무 시나리오를 수행한다. 주요 평가지표는 과제 완료 시간, 필수 정보 누락률, 재작업 횟수, 주관적 사용성 점수이다. 본 연구는 통합 조회 플랫폼이 단순 기능 시연을 넘어 실제 조사 워크플로우의 효율성과 일관성을 얼마나 개선하는지 정량적으로 제시하는 것을 목표로 한다. 결과는 화학안전 실무에서 통합 정보 시스템의 도입 타당성을 평가하는 근거로 활용될 수 있다.

#### 3) 섹션 구성
1. 서론
   - 문제 배경: 분절된 화학안전/규제 조회 업무
   - 기존 한계: 시간 지연, 누락, 편차
   - 연구 목적과 질문
2. 관련 연구
   - 화학 정보 검색 플랫폼
   - 실무형 사용자 평가 연구
   - 정보 통합 시스템의 업무 효율 평가
3. ChemIP 개요
   - 시스템 구성
   - 통합 조회 대상
   - 기존 방식 대비 차이점
4. 연구 설계
   - 참가자 구성
   - 시나리오 정의
   - 비교 조건(기존 방식 vs ChemIP)
   - 평가지표
5. 결과
   - 시간 비교
   - 누락률/재작업률 비교
   - 사용자 만족도
6. 논의
   - 어떤 과제에서 효과가 큰가
   - 초보자/숙련자 차이
   - 시스템 도입 시 기대효과와 한계
7. 결론
   - 실무적 의미
   - 향후 확장 계획

#### 4) 실험 설계 초안
- 참가자: 6~12명
  - 실무자, 대학원생, 연구보조 인력 등 혼합 가능
- 시나리오: 3~5개
  1. 수출 대상 물질 사전검토
  2. 해외기업 국내 진입 사전점검
  3. 의약품 원료 교차참조
  4. 특허 위험 검토
  5. 규제 분류 확인
- 비교 방식
  - A군: 기존 포털 직접 조회
  - B군: ChemIP 사용
  - 가능하면 crossover 설계(순서 바꿔서 수행)
- 측정 지표
  - 완료 시간(분)
  - 필수 정보 누락 수
  - 재검색/재작업 횟수
  - 정답률 또는 task completion quality
  - SUS 또는 5점 리커트 만족도
- 분석
  - 참가자별 paired comparison
  - median, IQR, effect size 중심
  - 표본이 작으면 비모수 검정 사용

#### 5) 필요한 데이터/표/그림 목록
- 표 1: 참가자 구성
- 표 2: 시나리오 설명 및 요구 정보 목록
- 표 3: 정량 결과 요약(시간, 누락률, 재작업률)
- 그림 1: 평가 실험 절차도
- 그림 2: 기존 방식 vs ChemIP 방식 시간 비교 boxplot
- 그림 3: 누락률/재작업률 비교 bar chart
- 부록: 평가 지침서, 체크리스트, 설문 문항

#### 6) 바로 해야 할 일
- 대표 시나리오 3개 확정
- 각 시나리오별 "필수 확인 정보" gold checklist 작성
- 참가자 모집 기준 정의
- 측정 템플릿(시간, 누락, 재작업, 만족도) 작성

### B. 데이터 품질/정규화 논문 구체화

#### 1) 제목 후보
1. **한국어·영어 혼합 화학물질 질의의 정규화가 검색 성능에 미치는 영향: ChemIP 사례 연구**
2. **화학물질명 변형과 동의어 문제를 해결하기 위한 한국어 중심 엔터티 정규화 파이프라인**
3. **CAS, 영문명, 한글명 간 불일치를 줄이기 위한 화학물질 검색 정규화 연구**
4. **다원적 규제 데이터 소스에서의 화학물질 엔터티 매칭: ChemIP의 정규화 접근**
5. **한국 화학안전 데이터 통합을 위한 화학물질명 정규화 및 검색 향상 기법**

#### 2) 초록 초안
화학물질 검색은 한글명, 영문명, 약칭, 상품명, CAS 번호 등 다양한 표기 체계가 혼재되어 있어 검색 실패와 정보 누락을 유발한다. 특히 한국 규제·안전 데이터와 국제 공개 데이터 소스를 함께 사용하는 경우, 동일 물질이 서로 다른 명칭으로 표현되어 통합 조회의 품질이 저하될 수 있다. 본 연구는 ChemIP 플랫폼을 사례로 하여 화학물질명 정규화 및 엔터티 매칭 파이프라인을 설계하고, 검색 성공률과 빈 결과율에 미치는 영향을 평가한다. 제안 방식은 한국어/영어 동의어 사전, CAS 기반 매핑, 표기 변형 규칙, 소스 간 우선순위 규칙을 결합한다. 평가를 위해 curated query set을 구축하고, 정규화 적용 전후의 retrieval recall, empty-result rate, top-k hit rate를 비교한다. 본 연구는 한국어 중심 화학안전 데이터 통합에서 검색 정규화의 중요성을 정량적으로 제시하고, 실무형 통합 플랫폼의 검색 품질 개선 방안을 제안한다.

#### 3) 섹션 구성
1. 서론
2. 문제 정의: 화학물질명 변형과 데이터 소스 간 불일치
3. 데이터셋 구축
4. 정규화 및 엔터티 매칭 방법
5. 실험 설정 및 평가 지표
6. 결과
7. 오류 분석
8. 결론 및 향후 과제

#### 4) 실험 설계 초안
- 데이터셋: 300~1000개 질의
- 질의 유형
  - 한글명
  - 영문명
  - 약칭/오탈자
  - CAS 번호
  - 기관별 상이 표기
- 비교군
  - baseline: 원문 질의 그대로 검색
  - method 1: 규칙 기반 정규화
  - method 2: 규칙+사전 기반 정규화
- 지표
  - top-1 hit rate
  - top-5 hit rate
  - empty-result rate
  - normalized entity resolution accuracy

#### 5) 필요한 데이터/표/그림 목록
- 표 1: 질의셋 구성
- 표 2: 소스별 명칭 불일치 예시
- 표 3: 정규화 전후 검색 성능
- 그림 1: 정규화 파이프라인 다이어그램
- 그림 2: 오류 유형 분포
- 부록: 동의어 규칙 예시, 정규화 사전 스키마

#### 6) 바로 해야 할 일
- 질의셋 후보 수집
- gold mapping 기준 정의
- 기관별 대표 케이스 추출
- baseline과 비교 실험용 스크립트 정리

### C. 증거 기반 안전요약 논문 구체화

#### 1) 제목 후보
1. **근거 추적 가능한 화학안전 요약 생성: ChemIP의 deterministic evidence synthesis 설계**
2. **LLM 의존을 줄인 화학안전 정보 통합 요약: 규칙 기반 근거 결합 접근**
3. **출처 기반 화학안전 요약 엔진의 설계와 평가: ChemIP 사례**
4. **설명가능한 화학안전 요약 생성을 위한 evidence bundle 기반 접근**
5. **다중 규제 소스의 근거 결합을 통한 화학안전 보고서 생성 프레임워크**

#### 2) 초록 초안
화학안전 정보 요약은 여러 출처에서 수집한 규제, 독성, 특허, 의약품 관련 정보를 통합해야 하므로 요약 과정의 근거 추적 가능성과 신뢰성이 중요하다. 그러나 대규모 언어모델 기반 요약은 환각, 출처 불명확성, 재현성 부족의 문제를 가질 수 있다. 본 연구는 ChemIP 플랫폼에서 구현 가능한 deterministic evidence synthesis 접근을 제안한다. 제안 방식은 다중 소스에서 수집한 구조화된 evidence bundle을 기반으로 출처별 우선순위, 충돌 해결 규칙, confidence scoring을 적용하여 안전요약을 생성한다. 또한 선택적 LLM 보조와 기본 deterministic 경로를 분리하여 안전 분야에서의 설명가능성과 재현성을 강화한다. 본 연구는 화학안전 정보 통합 환경에서 근거 기반 요약의 설계 원칙과 평가 기준을 제시한다.

#### 3) 섹션 구성
1. 서론
2. 배경: 안전요약과 설명가능성 문제
3. ChemIP evidence bundle 구조
4. deterministic synthesis 규칙 설계
5. confidence score 및 source attribution
6. 평가 설계
7. 결과 및 사례 분석
8. 논의와 한계
9. 결론

#### 4) 실험 설계 초안
- 입력: 대표 물질 30~100개
- 비교 조건
  - 규칙 기반 deterministic summary
  - 선택적 LLM-assisted summary
- 평가자
  - 준전문가 또는 전문가 3~5명
- 평가 항목
  - 근거 추적 가능성
  - 중요한 위험 정보 누락 여부
  - 요약의 일관성
  - 신뢰도
  - 재현성
- 정량/정성 혼합 평가
  - rubric scoring
  - 사례 기반 오류 분석

#### 5) 필요한 데이터/표/그림 목록
- 표 1: evidence bundle schema
- 표 2: 출처별 우선순위 규칙
- 표 3: 평가 결과 비교표
- 그림 1: evidence synthesis pipeline
- 그림 2: deterministic vs LLM-assisted 예시 비교
- 부록: confidence score 계산 규칙, 요약 템플릿

#### 6) 바로 해야 할 일
- evidence bundle 필드 정의
- 충돌 해결 규칙 문서화
- 요약 템플릿 설계
- 평가 rubric 초안 작성
