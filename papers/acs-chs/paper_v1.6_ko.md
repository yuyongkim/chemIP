# ChemIP: 한국 및 국제 규제 출처를 통합한 오픈소스 화학물질 안전 데이터 조회 웹 플랫폼

## 초록

화학물질의 규제 준수를 평가하는 안전 실무자는 각기 다른 인터페이스, 검색 구문, 응답 형식을 가진 여러 정부 포털을 개별적으로 조회해야 한다. 대한민국에서는 유해성 데이터, 물질 분류, 특허 현황, 의약품 등록 정보가 서로 연동되지 않는 별도 기관에서 관리된다. 이러한 분절된 워크플로우는 조사 시간을 증가시키고, 정보 누락 위험을 야기하며, 실무자 간 평가 품질의 불균일을 초래한다. ChemIP은 한국 및 국제 안전·규제·특허 데이터 출처를 단일 검색 워크플로우로 통합하는 오픈소스, 자체 호스팅 가능 웹 플랫폼이다. 핵심 기능은 GHS 분류를 포함한 물질안전보건자료(MSDS) 조회, 한국 물질 규제 분류 검색, 실시권 분석을 위한 특허 현황 조회, 의약품 교차참조, AI 기반 안전성 종합 분석을 포함한다. 본 플랫폼은 어댑터 기반 아키텍처와 우아한 성능 저하(graceful degradation) 설계를 채택하여 개별 데이터 출처가 이용 불가능한 경우에도 지속적인 운영을 보장한다. 예비 워크플로우 분석에 따르면 물질별 조사 시간이 중복 인증 및 수작업 데이터 편집 과정의 제거를 통해 약 150분에서 35분으로 단축될 수 있는 것으로 추정된다. 소스 코드, 자동화 테스트, CI/CD 구성은 공개 GitHub 저장소(https://github.com/yuyongkim/chemIP)에 호스팅되며, 소프트웨어의 아카이브 스냅샷은 Zenodo에 기탁되어 있다(DOI: 10.5281/zenodo.19447588).

## 서론

화학물질의 안전한 취급, 운송 및 거래를 위해 실무자는 여러 범주의 규제 및 안전 정보를 교차 참조해야 한다. 단일 물질에 대해 일반적으로 (1) GHS 그림문자, 신호어, H-진술문을 포함하는 물질안전보건자료(MSDS)의 유해성 분류, (2) 독성 지정 및 사고대비 요건 등 물질별 규제 분류, (3) 실시 자유도(freedom-to-operate) 분석을 위한 특허 현황, (4) 화학물질이 의약품 제형과 중복되는 경우의 의약품 규제 현황이 필요하다.

세계 화학 산업은 다수의 상업적 유통 물질을 포함한다. 국제화학공업협의회(ICCA)는 전 세계적으로 수만 종의 화학 제품이 활발히 상업적으로 사용되고 있다고 보고한다.^1 Eurostat 무역 통계에 따르면 가장 최근 이용 가능한 연도의 EU 화학제품 및 관련 제품 수출은 약 5,600억 유로 규모이다.^2 이러한 산업 규모는 효율적인 안전 데이터 조회 도구의 필요성을 뒷받침한다.

대한민국에서는 화학물질관리법^16과 산업안전보건법^17에 따라 화학물질 안전 책임이 6개 기관에 분산되어 있다: MSDS 및 직업적 노출 데이터를 관리하는 한국산업안전보건공단(KOSHA), 특허 검색을 담당하는 한국지식재산정보서비스(KIPRIS), 무역 정보를 제공하는 대한무역투자진흥공사(KOTRA), 의약품 등록을 관리하는 식품의약품안전처(MFDS), 그리고 환경부 산하 화학물질안전원(NICS)이 운영하는 화학물질정보시스템(KISCHEM)과 화학물질종합정보시스템(NCIS)이 물질 분류 및 노출 데이터를 담당한다. 각 기관은 고유한 검색 인터페이스와 응답 형식을 가진 독립적인 웹 포털을 운영한다.

일반적인 물질 평가에서 실무자는 각 포털에 순차적으로 접근하여 검색어를 재입력하고 결과를 수작업으로 취합해야 한다. 내부 도입 계획서에서는 이 워크플로우의 4가지 체계적 비효율성을 문서화하였다:^3 (1) 포털 간 연결이 없는 분절된 검색 채널, (2) 반복적인 수작업 조회에 의한 처리 지연, (3) 실무자 전문성 차이에 따른 불일치한 결과, (4) 초기 MSDS 검토 후 특허 및 시장 조사가 단절되는 후속 워크플로우 중단.

이러한 교차 참조 부담은 두 그룹에 특히 크다: 대상 물질이 수출국 규제를 충족하고 기존 특허를 침해하지 않는지 확인해야 하는 화학물질 수출 준비 중인 한국 중소기업(SME)과, 국내 유통 전에 한국 고유의 안전 분류 및 물질 제한을 파악해야 하는 한국 시장 진입 외국 기업이다.

핵심 문제는 공공 데이터의 부재가 아니라 — 정보는 잘 관리되는 정부 서비스에 걸쳐 존재한다 — 워크플로우의 분절이다: 이러한 출처를 통합된 검색 프로세스로 연결하는 공개 도구가 없다. 기존 플랫폼은 이 문제의 일부를 해결한다: PubChem^4은 포괄적인 화학물질 속성, GHS 유해성 분류 요약, 특허 추출 화학 구조를 제공하지만, 16개 섹션의 완전한 MSDS 문서나 한국 규제 포털 전반의 다중 출처 오케스트레이션을 제공하지 않는다. SciFinder^5와 Reaxys^6은 특허 통합 분자 검색을 제공하지만 한국 규제 데이터를 다루지 않는 상용 제품이다. OpenFDA^7는 교차 도메인 통합 없이 미국 의약품 안전을 다룬다. PubChem이 GHS 분류 데이터와 특허 검색 기능을 제공하고, SciFinder와 Reaxys가 의약품 교차참조를 제공하지만, 이 플랫폼들 중 어느 것도 한국 특화 규제 데이터 출처(KOSHA 전체 MSDS, KISCHEM, NCIS)를 특허 검색과 통합된 안전 평가 워크플로우로 연결하지 않는다. 2026년 초에 수행한 문헌 및 제품 문서 검토에 기반하여, 이러한 한국 규제 포털과 국제 출처에 걸친 실시간 쿼리를 단일 워크플로우로 오케스트레이션하는 공개 문서화된 오픈소스, 자체 호스팅 가능 플랫폼은 저자가 아는 한 존재하지 않는다.^4-7

기존 도구가 주로 화학 구조나 물성 검색에 초점을 맞추는 것과 달리, ChemIP은 한국 맥락에서 수출 대상 및 수입 물질의 규제 안전 평가를 위한 전 과정(end-to-end) 워크플로우를 중심으로 설계되었다. 본 논문은 안전, 규제 및 특허 데이터 출처를 단일 검색 화학물질 안전 조사 워크플로우로 통합하는 오픈소스 웹 플랫폼 ChemIP을 소개하고, 그 아키텍처, 안전 관련 기능, 예비 도입 경험을 기술한다.

## 플랫폼 설계

**아키텍처.** ChemIP은 3계층 아키텍처를 따른다: 브라우저 기반 인터페이스,^13 다수의 안전·규제·특허 데이터 출처에 대한 쿼리를 처리하는 Python 서버 계층(FastAPI^12), 오프라인 접근을 위한 로컬 데이터 캐시. 서버는 외부 데이터 출처, 로컬 데이터베이스, 그리고 안전 보고서 생성을 위한 선택적 로컬 언어 모델(Ollama^10)을 오케스트레이션한다. 유해성 분류는 세계조화시스템(GHS Rev. 10)을 따른다.^11

**어댑터 기반 통합.** 각 외부 데이터 출처는 인증, 요청 구성, 응답 파싱(XML 또는 JSON), 오류 정규화를 처리하는 전용 어댑터 모듈(표 1)로 래핑된다. 모든 어댑터는 정부 포털의 일시적 장애 시 자동 재시도 및 장애 조치를 수행하는 통신 계층을 공유하여 신뢰성 있는 안전 데이터 조회를 보장한다. 대규모 특허 분석이 필요한 도입 환경에서는 로컬 캐시된 특허 코퍼스^9에 대한 효율적 다중 키워드 검색^8을 사용하는 선택적 사전 구축 특허 인덱스를 지원한다.

**표 1.** ChemIP v1.1.1 기준 통합 데이터 출처 및 어댑터 특성.

| 출처 | 형식 | 범위 |
|------|------|------|
| KOSHA MSDS^14 | XML | GHS 분류를 포함한 16개 섹션 안전 데이터 (data.go.kr 대체 경로^15) |
| KIPRIS 특허 | XML | 한국 특허 검색 및 상세 조회 |
| KOTRA (7개 하위 API) | JSON/XML | 시장 뉴스, 진출 전략, 가격 정보, 무역 사기, 수입 규제 |
| MFDS | JSON | 한국 의약품 허가 데이터 |
| OpenFDA^7 | JSON | 미국 의약품 라벨 및 부작용 데이터 |
| PubMed | JSON | 생의학 문헌 검색 |
| Naver | JSON | 한국어 뉴스 검색 |
| KISCHEM | XML | 노출 증상 및 응급처치 데이터 |
| NCIS | JSON | 물질 분류 (유독물질, 제한물질, 사고대비물질) |

**우아한 성능 저하(Graceful degradation).** 본 시스템은 개별 구성요소가 이용 불가능할 때에도 작동하도록 설계되어 있으며, 이는 신뢰성 있는 안전 정보 조회에 중요하다. KOSHA API가 중단되면 로컬 데이터베이스에서 캐시된 MSDS 데이터를 제공한다. KOSHA MSDS 어댑터는 대체 메커니즘을 구현한다: 기본 API 엔드포인트(msds.kosha.or.kr)^14에 접근할 수 없으면, 공식 KOSHA MSDS 공개 API 문서에 기술된 대로 요청이 자동으로 data.go.kr 공공데이터포털 프록시 엔드포인트^15로 전환된다. 선택적 특허 인덱스 데이터베이스가 설치되어 있지 않으면, 특허 검색은 실시간 KIPRIS API 쿼리로 대체된다. LLM 서버(Ollama)가 오프라인이면, 결정론적 규칙 기반 보고서 빌더가 출처 귀속이 포함된 구조화된 안전 요약을 생성한다.

**화학물질 안전 기밀성을 위한 자체 호스팅 도입.** 본 플랫폼은 로컬 도입을 지원하여 독점 제형이나 특허 출원 전략을 드러낼 수 있는 화학물질 조회 패턴이 조직 네트워크 내에 유지되도록 한다. 이는 수출 후보 물질을 평가하는 중소기업과 시장 진입 전 규제 준수 검토를 수행하는 기업에 특히 중요하며, 물질 평가 패턴은 영업 기밀 정보에 해당할 수 있다. 공개 저장소 README에 단계별 설치 가이드와 예시 구성 파일이 제공되어 실무자가 자체 환경에서 도입을 재현할 수 있다.

## 안전 관련 기능

ChemIP은 탭 인터페이스(그림 1)를 통해 5가지 핵심 안전 워크플로우를 제공한다:

1. **물질 안전 조회.** 단일 검색(화학물질명, CAS 번호 또는 영문명)으로 GHS 그림문자, 유해성 진술문(H-코드), 예방 조치 진술문(P-코드), 신호어를 포함한 16개 MSDS 섹션 전체를 조회하며, 최초 조회 후 로컬에 캐시된다.

2. **한국 규제 분류.** KISCHEM 노출 증상 및 응급처치 데이터와 NCIS 물질 분류(유독물질, 제한물질, 사고대비물질) 및 분자 데이터를 통합한다. 한국 규제가 시장 진입 전에 요구하는 정보를 통합 제공한다.

3. **특허 현황.** 실시간 KIPRIS API 쿼리와 선택적 로컬 인덱스 검색을 결합하는 이중 트랙 검색으로, 수출 대상 물질의 실시 자유도(freedom-to-operate) 분석을 지원한다. 규제 안전 데이터와 특허 현황을 단일 워크플로우에서 연결함으로써, 실무자가 포털 간 전환 없이 규제 준수 요건과 지적재산권 제약을 동시에 파악할 수 있다.

4. **의약품 교차참조.** 의약품 제형과 중복되는 물질에 대해 한국 의약품 허가(MFDS), 미국 의약품 라벨(OpenFDA), 생의학 문헌(PubMed)을 통합 제공한다.

5. **AI 기반 안전성 종합 분석.** 안전, 규제, 특허 영역에 걸쳐 증거를 수집하고 자연어 안전 요약을 생성한다. LLM이 이용 불가능한 경우, 규칙 기반 대체 모듈이 출처 귀속과 신뢰도 점수가 포함된 구조화된 보고서를 생성한다.

본 플랫폼은 추가로 보조적인 무역 정보(시장 뉴스, 진출 전략, KOTRA API를 통한 수입 규제)와 한국어 뉴스 검색을 제공하여 핵심 안전 평가를 넘어선 폭넓은 수출 사전 심사 워크플로우를 지원한다.

**안전 가이드 추천.** 다중 기준 엔진이 CAS 번호, 제목 키워드, 문서 본문에 걸친 가중 정확 일치 점수를 사용하여 화학물질-가이드 매칭을 평가하며, KOSHA 안전 가이드 추천의 재현 가능한 순위를 위한 결정론적 랭킹을 적용한다.

## 화학 안전 실무를 위한 사용 사례

다음 시나리오는 ChemIP이 다중 포털 워크플로우를 통합 검색으로 축약하는 방식을 설명하며, 내부 도입 계획서의 문서화된 사용 사례에서 도출하였다.^3 이는 대표적인 사용 예시이며 공식 평가가 아니다.

**중소기업 수출 사전 심사.** 한국 중소기업이 톨루엔(CAS: 108-88-3)을 공업용 용제로 베트남에 수출하려 한다. 선적 전 규제 준수 담당자는 유해성 분류(GHS 그림문자 GHS02, GHS07, GHS08; TWA: 50 ppm)와 특허 실시 자유도를 확인해야 하며, 이 정보는 KOSHA와 KIPRIS 포털에 분산되어 있다. 단일 ChemIP 검색으로 GHS 분류를 포함한 16개 섹션 전체 MSDS, 실시 자유도 분석을 위한 KIPRIS 특허 결과, NCIS 규제 분류, 그리고 매칭된 KOSHA 안전 가이드 3건을 조회한다. 보조적 무역 데이터(수입 규제, 진출 전략)는 추가 탭에서 이용할 수 있다. 기존 워크플로우는 각각 별도 인증이 필요한 다수의 포털 접근을 요구하나, ChemIP은 안전, 규제, 특허 데이터를 하나의 인터페이스로 통합한다.

**외국 기업의 한국 시장 진입.** 다국적 화학물질 유통업체가 한국 내 유통을 위해 메탄올(CAS: 67-56-1)을 평가한다. 한국 규제는 시장 진입 전에 물질별 안전 분류(NCIS), 노출 증상 데이터(KISCHEM), MSDS 준수(KOSHA)를 요구하며 각각 다른 기관이 관리한다. 단일 ChemIP 검색으로 NCIS 분류(유독물질, 사고대비물질), KISCHEM 응급처치 및 노출 데이터, 전체 KOSHA MSDS, MFDS 의약품 교차참조를 조회하여, 규제 준수 팀이 한 세션에서 모든 한국 고유 안전 요건을 파악할 수 있다.

**의약품 원료 안전성 교차참조.** 원료 공급업체가 거래처의 규제 우려 보고 후 살리실산(CAS: 69-72-7)을 조사한다. 의약품 워크플로우가 MFDS 허가 제품, 관련 경고를 포함한 OpenFDA 라벨, 최근 PubMed 논문을 MSDS 자극성 분류와 함께 통합 제공하여, 별도 3회 검색이 필요한 한국 및 미국 규제 데이터베이스를 하나로 축약한다.

## 결과 및 논의

**예비 워크플로우 분석.** 3가지 표준 평가 시나리오(톨루엔 수출 심사, 메탄올 시장 진입, 살리실산 교차참조)에 기반한 내부 워크플로우 분석에 따르면, 물질별 조사 시간이 약 150분(순차적 다중 포털 접근 및 별도 인증)에서 약 35분(단일 검색 통합 조회)으로 감소하여 약 76%의 잠재적 단축이 추정된다.^3 개별 워크플로우 단계 측면에서, 별도 포털 접근 횟수가 최소 4회(각각 별도 인증)에서 1회로 줄어들며, 포털 간 수작업 결과 편집이 완전히 제거된다. 이 수치는 저자가 이 3가지 대표 사례를 사용한 스크립트 기반 시나리오 워크스루에 기반하며, 통제된 시간-동작 연구가 아니므로 참고 수치로 해석해야 한다. 이 값들은 통계적으로 검증된 성능 지표가 아닌, 잠재적 시간 절감을 예시하기 위한 대략적 규모 추정치임을 강조한다. 현장 실무자를 대상으로 한 공식 시간-동작 연구가 향후 과제로 계획되어 있다.

**도입 경험.** 2일간의 내부 도입 연습(2026년 2월) 중 여러 문제가 식별 및 해결되었다: (1) GHS 유해성 진술문 수집을 영문명 매칭 이상의 범위를 확보하기 위해 배치 파이프라인으로 전환, (2) 기본 엔드포인트가 불완전한 데이터를 반환하는 엣지 케이스에 대한 KOSHA MSDS 대체 로직 개선, (3) 유독물질, 제한물질, 사고대비물질 범주를 명확히 구분하도록 NCIS 분류 표시 재구조화.^3 이러한 개선은 어댑터 기반 아키텍처가 안전 정보 조회에서 발생하는 실제 데이터 품질 문제에 대응하여 신속한 반복 개선을 지원함을 보여준다. 이러한 관찰은 체계적 유해성 인식과 위험성 평가를 화학 안전 실무의 기반으로 강조하는 ACS의 광범위한 지침과 일치한다.^18

**소프트웨어 품질.** 저장소는 모의(mock) 외부 의존성을 사용하는 단위 수준 스모크 테스트를 포함하며, 모든 API 엔드포인트 범주를 커버한다. 테스트 스위트는 외부 서비스에 접속하지 않는 격리 환경에서 실행되며, ChemIP v1.1.1 기준으로 전체 스위트가 통과한다. 자동화된 품질 검사가 매 코드 업데이트 시 실행된다. API 엔드포인트 문서는 플랫폼의 내장 문서 인터페이스를 통해 이용할 수 있다.

**도입 모드.** ChemIP은 3가지 도입 구성을 지원한다: (1) 최소(Minimal) — 용어 데이터베이스와 실시간 API 호출을 사용한 핵심 검색 및 MSDS 조회, data.go.kr API 키만 필요; (2) 표준(Standard) — 캐시된 MSDS 데이터, 가이드 추천, 의약품 교차참조, 한국 규제 조회 추가; (3) 전체(Full) — 선택적 로컬 특허 인덱스 및 LLM 지원 분석 추가.

**기존 도구와의 비교.** 표 2는 2026년 초 기준 공개 제품 문서에 기반하여 ChemIP과 기존 플랫폼을 비교한다. 이 비교는 포괄적이기보다 참고적인 조사로 해석해야 한다.

**표 2.** 화학물질 안전 정보 플랫폼 기능 비교. Y = 지원, partial = 제한적 지원, -- = 미지원. "Full MSDS"는 GHS/OSHA 형식의 구조화된 16개 섹션 MSDS 문서, "GHS classification data"는 유해성 그림문자, H-코드, 신호어를 의미한다. 2026년 초 기준 공개 제품 문서 기반 비교.

**한계.** 공식 사용자 연구는 아직 수행되지 않았으며, 현장 실무자 대상 사용성 평가가 계획되어 있다. 일부 워크플로우는 사양 변경이나 장애가 발생할 수 있는 외부 정부 API에 의존하며, 개발 중 간헐적 KOSHA 서비스 중단이 관찰되었다. 어댑터 기반 설계가 대체 메커니즘을 통해 이를 완화한다. 로컬 캐시된 MSDS 데이터는 상위 기관이 기록을 업데이트하면 오래될 수 있으며, 현재 구현에는 자동 캐시 무효화 메커니즘이 포함되어 있지 않아 캐시된 안전 데이터가 최신 규제 변경을 반영하지 않을 수 있음을 사용자가 인지해야 한다. 한국 규제 데이터 출처(KOSHA, KISCHEM, NCIS)는 주로 한국어로 데이터를 반환하며, 플랫폼이 일부 MSDS 필드에 이중 언어 표시를 제공하지만 비한국어 사용자는 한국 규제 분류 모듈의 활용도가 제한적일 수 있다. LLM 구성요소(Ollama를 통한 Qwen3:8b)는 범용 모델이며, 도메인 특화 파인튜닝이 생성된 안전 분석의 관련성을 향상시킬 수 있다. 전체 도입에는 한국 정부 데이터 포털(data.go.kr)의 API 키가 필요하나, 키 없이도 플랫폼은 축소 모드로 운영된다. 본 플랫폼은 정부 공개 API의 데이터를 각 이용약관에 따라 집계 및 표시하며, 재배포 또는 상업적 이용 시 각 기관의 데이터 이용 정책을 참조해야 한다.

## 결론

ChemIP은 안전, 규제, 특허, 의약품 데이터 출처를 화학 안전 실무자를 위한 단일 검색 워크플로우로 통합하는 오픈소스 어댑터 기반 통합 플랫폼을 제공한다. 본 플랫폼은 한국 중소기업과 해외 시장 진입 기업이 물질 평가를 위해 다수의 분리된 정부 포털을 탐색해야 하는 워크플로우 분절 문제를 해결한다. 자체 호스팅 아키텍처는 화학물질 조회 패턴의 기밀성을 보존하고, 우아한 성능 저하 설계는 개별 데이터 출처가 이용 불가능할 때에도 지속적인 운영을 보장한다. 향후 과제로는 현장 실무자 대상 공식 사용성 연구, 추가 규제 관할권 확장, 안전 분석을 위한 도메인 특화 LLM 파인튜닝 탐구가 포함된다. 소스 코드는 AGPL-3.0 라이선스 하에 https://github.com/yuyongkim/chemIP에서 이용할 수 있으며, 아카이브 스냅샷은 Zenodo에 기탁되어 있다(DOI: 10.5281/zenodo.19447588).

## 감사의 글

본 연구는 한국산업안전보건공단(KOSHA), 한국지식재산정보서비스(KIPRIS), 대한무역투자진흥공사(KOTRA), 식품의약품안전처(MFDS), 그리고 환경부 산하 화학물질안전원(NICS)이 운영하는 KISCHEM 및 NCIS 서비스에서 제공하는 공개 API를 활용하였다. 서드파티 통합을 가능하게 하는 공개 데이터 서비스를 유지하는 각 기관에 감사한다. 본 원고는 Claude(Anthropic)의 도움을 받아 작성되었으며, 초안 작성, 편집, 코드 리뷰에 활용되었다.
