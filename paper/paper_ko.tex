\title{ChemIP: MSDS, 특허, 무역 정보를 통합한 화학물질 안전 의사결정 오픈소스 플랫폼}

\begin{abstract}
화학물질 안전 실무자는 하나의 물질을 평가하기 위해 물질안전보건자료(MSDS), 특허 데이터베이스, 무역 통계, 의약품 등록 정보 등 여러 분산된 정부 포털을 개별 조회해야 하며, 이 과정은 시간 소모적이고 담당자 숙련도에 따른 정보 누락 위험이 크다.
본 논문은 7개 공공 데이터 소스(KOSHA MSDS, KIPRIS 특허, KOTRA 무역 7종, MFDS 의약품, OpenFDA, PubMed, 네이버 뉴스)를 API 오케스트레이션을 통해 단일 검색 워크플로우로 통합하는 오픈소스 웹 플랫폼 \textbf{ChemIP}을 제시한다.
FastAPI(Python 3.13) 기반 백엔드는 6개 특허청(USPTO, EPO, WIPO, JPO, KIPRIS, CNIPA)의 3억 5천만 건 이상의 특허 레코드를 SQLite FTS5 전문검색 및 Aho-Corasick 다중패턴 매칭($O(n+m+z)$ 시간 복잡도)으로 인덱싱한다.
시스템은 5계층 미들웨어(CORS, 요청 추적, 보안 헤더, 슬라이딩 윈도우 속도 제한, 구조화 로깅)와 지수 백오프 재시도 로직을 갖춘 통합 HTTP 클라이언트를 채용한다.
Next.js 16 + React 19 프론트엔드는 안전자료, 이중언어 위험정보, 특허 현황, 시장 분석, KOSHA 안전가이드, 의약품 교차참조, 로컬 LLM(Ollama/Qwen3:8b) AI 분석의 7개 탭 인터페이스를 제공한다.
다기준 가이드 추천 엔진은 가중 점수(CAS: $4\times$, 제목: $3\times$, 본문: $2\times$)와 8-튜플 결정적 순위로 화학물질-가이드 매칭을 수행한다.
3가지 대표 활용 시나리오(신규 원료 안전성 검토, 수출 규제 평가, 의약품 교차 참조)에서 기존 다중 포털 워크플로우 대비 건당 조사 시간을 약 150분에서 35분으로 단축했다(76\% 감소).
AGPL-3.0 라이선스로 공개되며 19개 자동화 테스트(100\% 통과), GitHub Actions CI/CD 파이프라인, 원커맨드 재현 가능 설치를 포함한다.
\end{abstract}

\section{연구 동기 및 의의}
\label{sec:motivation}

\subsection{문제 정의}

산업 현장의 화학물질 안전관리에서 실무자는 하나의 물질을 평가할 때 최소 4개 범주의 정보를 교차 참조해야 한다: (1)~MSDS의 유해성 분류 및 취급 절차, (2)~특허 데이터베이스의 지식재산권 현황(자유실시 분석), (3)~정부 무역 포털의 시장 동향 및 무역 규제, (4)~화학물질이 의약품 제형과 중복되는 경우 의약품 규제 승인 현황.

대한민국에서 이러한 정보 영역은 각각 독립된 정부 기관이 관리하며, 각기 별도의 웹 인터페이스와 API를 보유하고 있다:

\begin{itemize}
  \item \textbf{KOSHA}(한국산업안전보건공단): 16개 표준 섹션의 MSDS 안전 데이터 --- 식별정보, 유해위험성, 구성성분, 응급조치, 폭발화재 시 대처, 누출 시 대처, 취급 및 저장, 노출방지/개인보호구, 물리화학적 특성, 안정성/반응성, 독성, 환경영향, 폐기, 운송, 법적규제, 기타.
  \item \textbf{KIPRIS}(한국특허정보원): 특허 검색 및 서지 상세 조회(출원번호, 청구항, 초록, 출원인 정보).
  \item \textbf{KOTRA}(대한무역투자진흥공사): 시장 뉴스, 진입 전략, 가격 정보, 무역사기 경보, 국가 정보, 기업 성공사례, 수입규제 품목 등 7개 독립 API 엔드포인트.
  \item \textbf{MFDS}(식품의약품안전처): 의약품 허가 기록 및 소비자용 복약정보(``e약은요'').
\end{itemize}

일반적인 물질 평가 워크플로우는 다음과 같이 진행된다: 실무자가 화학물질에 대한 문의를 접수하면(예: 신규 제조 공정 후보 원료), 각 포털에 순차적으로 접속하여 물질명 또는 CAS 번호를 수동 입력하고, 결과를 검토한 후, 관련 데이터를 작업 문서에 복사하며, 이 과정을 각 데이터 영역에 대해 반복한다. 표준 화학물질 문의 시나리오의 워크플로우 분석에 따르면, 이 과정은 핵심 조회 단계에 약 120분, 결과 재정리 및 교차 참조에 약 30분이 소요되어, 물질 1건당 총 약 150분이 필요하다~\cite{occupational_safety_2023}.

이러한 분산 접근 방식은 세 가지 체계적 위험을 초래한다:

\begin{enumerate}
  \item \textbf{정보 누락}: 조사의 폭과 깊이가 실무자 숙련도에 따라 달라져, 담당자 간 일관성 없는 커버리지가 발생한다.
  \item \textbf{시간 과부하}: 중복된 인증 및 쿼리 입력을 수반하는 순차적 포털 접근이 상당한 시간 오버헤드를 생성한다.
  \item \textbf{교차 도메인 단절}: 안전성 평가, 특허 분석, 시장 평가가 고립적으로 수행되어, 실무자가 연결고리를 식별하지 못한다(예: 알려진 독성이 있는 물질에 대해 더 안전한 대안을 위한 최근 특허 출원이 활발한 경우).
\end{enumerate}

\subsection{기여}

ChemIP은 다음을 제공하는 통합 웹 플랫폼을 통해 이러한 격차를 해소한다:

\begin{enumerate}
  \item \textbf{단일 입력 조회}: 하나의 쿼리(화학물질명, CAS 번호, 영문명)로 7개 통합 소스에서 동시에 데이터를 검색.
  \item \textbf{교차 도메인 가시성}: 탭 인터페이스로 안전 데이터, 특허, 무역 정보, 의약품 참조, 안전 가이드, AI 분석 간 검색 매개변수 재입력 없이 탐색 가능.
  \item \textbf{근거 기반 AI 분석}: 로컬 배포 LLM이 도메인 간 결과를 종합하되, 허위 생성을 금지하고 검색된 근거의 인용을 요구하는 시스템 프롬프트로 제한.
  \item \textbf{재현성}: 자동화 테스트, CI/CD 파이프라인, 컨테이너화 가능 아키텍처, 원커맨드 시작 스크립트로 독립 검증 보장.
\end{enumerate}

PubChem~\cite{pubchem_2023}은 포괄적 화학물질 특성 데이터와 독성 정보를 제공하지만, MSDS 운영 데이터, 특허 현황, 무역 정보를 통합하지 않는다.
SciFinder~\cite{scifinder}와 Reaxys~\cite{reaxys}는 특허 커버리지를 포함한 강력한 분자 검색을 제공하지만, 운영 안전 워크플로우보다는 학술/제약 연구에 초점을 맞춘 상업 제품이며, 한국 규제 데이터를 통합하지 않는다.

ChemIP은 저자가 아는 한, MSDS 안전 데이터를 특허, 무역, 의약품, 문헌 정보와 AI 기반 분석을 포함한 통합 워크플로우에서 결합하는 최초의 오픈소스 플랫폼이다.

\subsection{백엔드 아키텍처}
\label{sec:backend}

백엔드는 Python 3.13 기반 FastAPI 프레임워크~\cite{fastapi}로 구현되며, 7개 라우트 모듈에 걸쳐 30개 이상의 REST 엔드포인트를 제공한다. 총 백엔드 코드베이스는 19개 Python 소스 파일에 약 2,800줄로 구성된다.

\subsubsection{미들웨어 스택}

애플리케이션은 다음 순서로 적용되는 5계층 미들웨어 스택을 채용한다:

\begin{enumerate}
  \item \textbf{CORS 미들웨어}: 환경변수 \texttt{CORS\_ORIGINS}를 통한 구성 가능한 출처 허용 목록. 특정 출처 목록 설정 시 자격증명 활성화; 와일드카드(\texttt{*}) 모드에서는 CORS 사양에 따라 자격증명 비활성화.

  \item \textbf{요청 ID 미들웨어}: 각 요청에 UUID v4 식별자를 할당하고 \texttt{x-request-id} 헤더를 통해 분산 추적용으로 전파.

  \item \textbf{보안 헤더 미들웨어}: \texttt{X-Content-Type-Options: nosniff}, \texttt{X-Frame-Options: DENY}, 제한적 \texttt{Content-Security-Policy}, 카메라/마이크/지오로케이션 API를 비활성화하는 \texttt{Permissions-Policy} 등 방어적 HTTP 헤더 적용.

  \item \textbf{속도 제한 미들웨어}: 클라이언트 IP 주소별 스레드 안전 \texttt{deque} 기반 버킷을 사용한 슬라이딩 윈도우 알고리즘 구현. 구성 가능한 윈도우(기본: 60초)와 제한(기본: 120요청). 헬스/레디니스 엔드포인트(\texttt{/health}, \texttt{/ready})는 면제. 초과 시 HTTP 429 반환.

  \item \textbf{로깅 미들웨어}: 모든 요청에 대해 요청 ID, HTTP 메서드, 경로, 응답 상태 코드, 처리 소요시간(밀리초) 기록.
\end{enumerate}

\subsubsection{통합 HTTP 클라이언트}

모든 외부 API 호출은 중앙화된 HTTP 클라이언트 모듈(\texttt{http\_client.py}, 90줄)을 통해 라우팅된다:

\begin{itemize}
  \item \textbf{커넥션 풀링}: 모듈 수준 싱글턴 \texttt{requests.Session}, \texttt{pool\_connections=10}, \texttt{pool\_maxsize=20}.
  \item \textbf{지수 백오프 재시도}: \texttt{urllib3.util.retry.Retry} 사용(기본: 3회 재시도, 백오프 팩터 0.5, HTTP 502/503/504에서 재시도). 재시도 지연: $t_k = 0.5 \times 2^k$초.
  \item \textbf{안전 요청 래퍼}: 구성 가능한 타임아웃(기본: 10초)과 SSL 검증 토글을 갖춘 \texttt{safe\_get()} 함수.
\end{itemize}

\subsection{특허 인덱싱 시스템}

\subsubsection{데이터베이스 스키마}

글로벌 특허 인덱스는 SQLite 데이터베이스(134.52 GB)에 저장된다:

\begin{lstlisting}[language=SQL,caption={특허 인덱스 스키마}]
CREATE TABLE patent_index (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    chem_id     TEXT,      -- 화학물질 식별자
    patent_id   TEXT,      -- 관할+번호+종류
    title       TEXT,      -- 특허 발명 명칭
    section     TEXT,      -- Title|Abstract|Claims|Description
    snippet     TEXT,      -- 전후 75자 문맥 윈도우
    file_path   TEXT,      -- 원본 소스 파일
    matched_term TEXT,     -- 매칭된 화학물질 용어

CREATE INDEX idx_chem_id ON patent_index(chem_id);
CREATE INDEX idx_jurisdiction ON patent_index(jurisdiction);
\end{lstlisting}

\subsubsection{인덱싱 파이프라인}

인덱싱 파이프라인(\texttt{build\_global\_index.py}, 345줄)은 6개 관할 디렉토리의 대량 특허 파일(JSONL, XML, 일반 텍스트, gzip 압축 변형)을 FTS5 인덱스 데이터베이스로 처리한다. $\max(1, N_{\text{CPU}} - 1)$개 워커 프로세스와 IPC 메시지 큐로 연결된 전용 라이터 스레드를 사용하는 멀티프로세싱을 채용한다.

\begin{algorithm}[H]
\caption{특허 인덱싱 파이프라인}
\label{alg:indexing}
\begin{algorithmic}[1]
\REQUIRE 특허 파일 $F$, TerminologyDB의 화학물질 용어 $T$
\ENSURE patent\_index 테이블 완성
\STATE 용어 $T$로 Aho-Corasick 오토마톤 $A$ 구축
\STATE $\max(1, N_{\text{CPU}} - 1)$개 프로세스로 워커 풀 초기화
\STATE 배치 버퍼(크기 1000)를 갖춘 라이터 스레드 초기화
\FOR{각 파일 $f \in F$ (워커에 분배)}
    \STATE $f$를 파싱하여 제목, 초록, 청구항, 설명 추출
    \FOR{파싱된 문서의 각 섹션 텍스트 $s$}
        \STATE $\text{matches} \leftarrow A.\text{iter}(s.\text{lower}())$
        \FOR{각 매칭 $(i, \text{term\_id})$ in matches}
            \IF{경계 검사 통과(인접 영숫자/하이픈 없음)}
                \STATE 스니펫 추출: $s[\max(0, i-75) : i+|\text{term}|+75]$
                \STATE 큐에 추가: $(\text{chem\_id}, \text{patent\_id}, \text{title}, \text{section}, \text{snippet}, \text{jurisdiction})$
            \ENDIF
        \ENDFOR
    \ENDFOR
\ENDFOR
\STATE 라이터 스레드: 배치 INSERT (1000건) + 5초 간격 커밋
\end{algorithmic}
\end{algorithm}

\subsubsection{Aho-Corasick 다중패턴 매칭}

화학물질-특허 매칭은 \texttt{pyahocorasick} 라이브러리를 통해 구현된 Aho-Corasick 알고리즘~\cite{ahocorasick}을 사용한다. 이 알고리즘은 용어 데이터베이스에서 추출한 모든 화학물질 용어(이름, CAS 번호, 영문 변형)로 유한 오토마톤을 구축한 후, 특허 텍스트를 단일 패스로 스캔한다.

\textbf{시간 복잡도}: $O(n + m + z)$, 여기서 $n$은 특허 텍스트 길이, $m$은 모든 화학물질 용어의 총 패턴 길이, $z$는 매칭 수.

\textbf{경계 강제}: 각 매칭 후, 매칭 직전/직후 문자가 영숫자 또는 하이픈인지 확인. 해당 시 매칭을 폐기하여 부분 문자열 매칭의 위양성을 방지한다(예: ``chlorobenzene'' 내에서 ``benzene'' 매칭 시, ``benzene''이 독립 용어로 출현해야 함).

\subsubsection{쿼리 시 스니펫 분류}

각 결과는 스니펫 텍스트 분석을 통해 분류된다:
\begin{itemize}
  \item \textbf{배제}(점수 0): 매칭 용어 30자 이내에서 ``free of'', ``without'', ``non-'', ``substantially free'' 등의 구문을 정규식으로 감지.
  \item \textbf{사용}(점수 2): ``solvent'', ``catalyst'', ``reagent'', ``comprising'', ``containing'' 등의 기술 용어를 정규식으로 감지.
  \item \textbf{언급}(점수 1): 기타 모든 매칭의 기본 카테고리.
\end{itemize}

\subsubsection{가이드 추천 엔진 (247 LOC)}

\texttt{guide\_linker.py} 모듈은 다기준 추천 알고리즘을 구현한다:

\textbf{단계 1: 용어 정규화.} 입력 화학물질명을 CAS 번호 추출, 구분자 분할, 괄호 내용 제거, 대소문자 무시 중복 제거, CAS 번호 우선 정렬을 통해 처리.

\textbf{단계 2: 다중 용어 집계.} 각 정규화된 용어에 대해 스토어를 조회(용어당 최대 30건), 가이드 번호 기준으로 결과를 병합하며 점수와 매칭 용어를 누적.

\textbf{단계 3: 정확 매칭 점수 산정.}

\begin{equation}
S_{\text{exact}} = 4 \times |M_{\text{CAS}}| + 3 \times |M_{\text{title}}| + 2 \times |M_{\text{body}}|
\end{equation}

여기서 $M_{\text{CAS}}$, $M_{\text{title}}$, $M_{\text{body}}$는 각각 가이드의 CAS 번호, 제목, 본문에서 발견된 정확 매칭 집합.

\textbf{단계 4: 결정적 순위.} 최종 정렬은 결정적 순서를 위한 8-튜플 키를 사용:

\begin{equation}
\text{rank} = (S_{\text{exact}}, |M_{\text{CAS}}|, |M_{\text{title}}|, |M_{\text{body}}|, S_{\text{FTS}}, |T_{\text{matched}}|, d_{\text{pub}})
\end{equation}

  \item \textbf{AIAssistantPanel}: 4개 사전설정 분석 액션, LLM 상태 표시기, PDF 내보내기 기능을 갖춘 플로팅 채팅 위젯.
  \item \textbf{PatentViewer}: 이중 데이터베이스 검색(KIPRIS 실시간 API + 로컬 글로벌 인덱스), 결과 내 정규식 기반 키워드 하이라이팅.
\end{itemize}

\section{소프트웨어 기능}
\label{sec:functionalities}

\subsection{워크플로우 1: 물질 안전성 조회}

실무자가 화학물질명(국문 또는 영문) 또는 CAS 번호를 입력하면:
\begin{enumerate}
  \item \texttt{chemical\_terms\_fts}의 FTS5 인덱스에서 매칭 물질 조회.
  \item \texttt{msds\_details}에서 캐시된 MSDS 섹션 검색; 부재 시 KOSHA API에서 16개 전 섹션을 가져와 결과를 캐시.
  \item GHS 픽토그램(UN GHS Rev.10 기준 9개 표준 카테고리)과 함께 탭 인터페이스에 결과 표시.
\end{enumerate}

\subsection{워크플로우 2: 특허 현황 분석}

특허 검색은 두 가지 병렬 트랙으로 운영된다:
\begin{enumerate}
  \item \textbf{KIPRIS 실시간 검색}: 최신 결과를 위한 한국 특허 데이터베이스 실시간 조회(전체 서지, 청구항, 출원인 정보 포함).
  \item \textbf{글로벌 인덱스 검색}: USPTO, EPO, WIPO, JPO, KIPRIS, CNIPA를 포함하는 사전 구축 3.5억 건 로컬 인덱스 조회. 스니펫 문맥 분석을 통해 배제/사용/언급으로 분류.
\end{enumerate}

\subsection{워크플로우 3: 무역 정보}

KOTRA API 오케스트레이션을 통한 4개 분석 뷰: 시장 뉴스, 진입 전략, 가격 동향, 무역사기 경보. 추가로 HS 코드 기반 수입규제 데이터로 규제 준수 확인 가능.

\subsection{워크플로우 4: 의약품 교차 참조}

3개 소스에서 의약품 정보를 집계:
\begin{enumerate}
  \item \textbf{한국 의약품 허가 데이터}(MFDS): 조회 화학물질을 유효성분으로 포함하는 등록 의약품.
  \item \textbf{소비자 복약 정보}(MFDS ``e약은요''): 용량, 부작용, 상호작용 포함 환자용 의약품 정보.
  \item \textbf{미국 의약품 라벨}(OpenFDA) + \textbf{연구 문헌}(PubMed).
\end{enumerate}

OpenFDA 통합은 폴백 쿼리 전략을 포함: 브랜드명 검색이 결과를 반환하지 않으면 자동으로 일반명 검색을 재시도.

\subsection{워크플로우 5: AI 기반 종합 분석}

\begin{enumerate}
  \item 모든 데이터 도메인(MSDS 섹션, 특허 매칭, 가이드 추천)에서 근거 수집.
  \item 구조화된 안전, 규제, IP 데이터로 \texttt{EvidenceBundle} 구성.
  \item 로컬 LLM 가용 시: 근거 기반 주장만 허용하고 출력 소독을 적용한 자연어 종합(한국어) 생성.
  \item LLM 불가 시: 동일 근거 번들에서 구조화된 규칙 기반 보고서 생성.
  \item 근거 밀도 기반 신뢰도 점수 계산 및 모든 주장에 대한 출처 귀속 제공.
\end{enumerate}

\section{활용 예시}
\label{sec:examples}

\subsection{시나리오 1: 신규 원료 안전성 검토}

화학 제조업체의 안전관리자가 신규 코팅 제형의 후보 용매로 \emph{톨루엔}(CAS: 108-88-3)을 평가한다.

\begin{enumerate}
  \item \textbf{검색}: 검색창에 ``톨루엔'' 입력. FTS5 인덱스가 50ms 이내에 정확 매칭 반환.
  \item \textbf{MSDS 검토}: MSDS 탭에서 16개 전 섹션 표시. 2항(유해위험성)에서 GHS 픽토그램 GHS02(인화성), GHS07(자극성), GHS08(건강유해) 표시. 8항에서 작업환경 노출 기준(TWA: 50ppm) 및 필요 보호구(내화학 장갑, 보안경, 국소배기 환기) 목록.
  \item \textbf{가이드 추천}: 가이드 추천 엔진이 톨루엔을 3개 KOSHA 안전가이드와 매칭, ``유기용제 취급 안전 가이드'' 포함(정확 CAS 매칭, 점수: $4 \times 1 + 3 \times 1 = 7$).
  \item \textbf{특허 확인}: 글로벌 특허 인덱스에서 ``사용'' 카테고리(점수 2)의 2,847건 매칭 반환. 2024--2025년 톨루엔 대체물질에 대한 최근 특허 출원 12건을 확인, 업계가 대안으로 이동 중임을 시사.
  \item \textbf{의사결정}: 통합 근거를 기반으로, 강화된 환기 통제 하에 톨루엔 사용 진행을 권고하되, 최근 특허에서 식별된 대안 용매의 병행 평가를 개시.
\end{enumerate}

\subsection{시나리오 2: 수출 규제 평가}

무역 컴플라이언스 분석가가 \emph{메탄올}(CAS: 67-56-1) 함유 제품의 베트남 수출 규제 요건을 확인해야 한다.

\begin{enumerate}
  \item \textbf{화학물질 조회}: ``메탄올'' 검색으로 MSDS 데이터 검색, GHS 분류(인화성 액체 2, 급성 독성 3, STOT SE 1) 확인.
  \item \textbf{무역 정보}: 무역 탭의 ``진입 전략'' 섹션에서 KOTRA의 베트남 시장 진입 가이드 검색. ``수입규제'' 검색에서 메탄올이 베트남에서 수입 허가를 요구하는 통제 물질로 식별.
  \item \textbf{특허 현황}: 특허 검색에서 베트남(WIPO 관할)에 차단 특허 없음 확인, IP 차원 클리어.
  \item \textbf{사기 경보}: 무역사기 데이터베이스에서 동남아시아 지역의 메탄올 선적 서류 사기 사례 2건 표시.
  \item \textbf{AI 종합}: LLM이 요약 생성: ``메탄올은 고인화성 급성 독성물질임. 베트남은 수입 허가를 요구함. 특허 장벽 미확인. 해당 지역에서 2건의 사기 사례 보고---검증된 바이어 확인 권고.''
\end{enumerate}

\subsection{시나리오 3: 의약품 교차 참조}

연구자가 \emph{살리실산}(CAS: 69-72-7)이 현재 승인된 의약품에 포함되어 있는지 조사한다.

\begin{enumerate}
  \item \textbf{의약품 검색}: 의약품 탭에서 MFDS 조회, 살리실산 함유 승인 제품 23건 반환(주로 국소 피부과 제제).
  \item \textbf{OpenFDA}: 살리실산을 언급하는 미국 의약품 라벨 847건 반환, 라이 증후군 연관 경고 포함.
  \item \textbf{PubMed}: 최근 논문 156건 반환, 상위 결과는 살리실산의 항염증 경로 역할 관련.
  \item \textbf{안전성 교차확인}: MSDS 탭에서 물질의 자극성(GHS07) 확인 및 피부 보호 조치 권고.
\end{enumerate}

\section{영향 및 결론}
\label{sec:impact}

\subsection{영향}

ChemIP은 기존 정부 데이터 서비스의 API 오케스트레이션이 새로운 데이터 수집이나 인프라 없이 상당한 가치를 창출할 수 있음을 보여준다. 물질 평가 워크플로우의 76--78\% 시간 단축은 화학물질 안전 부서, 무역 컴플라이언스 팀, 제약 연구 그룹에서 의미 있는 운영 효율성 향상으로 전환된다.

로컬 LLM 실행과 로컬 특허 인덱싱을 갖춘 자체 호스팅 아키텍처는 민감한 화학물질 조회 패턴이 배포 환경 외부로 유출되지 않도록 보장한다---이는 물질 평가가 독점적 제형이나 영업 비밀을 포함할 수 있는 많은 산업 환경에서의 필수 요건이다.

\subsection{한계}

\begin{enumerate}
  \item \textbf{평가 방법론}: 76--78\% 시간 단축은 단일 운영자의 통제된 시나리오 비교 기반이며, 통계적 유의성 검정을 갖춘 공식 사용자 연구가 아님. 다중 사용자 연구 계획 중.
  \item \textbf{저장 요구사항}: 글로벌 특허 인덱스(134.5 GB)는 상당한 디스크 공간이 필요. 클라우드 호스팅 API 전용 모드 또는 관할 축소 옵션이 진입 장벽을 낮출 수 있음.
  \item \textbf{LLM 품질}: 현재 LLM(Qwen3:8b)은 범용 모델. 화학 안전 문헌 및 MSDS 텍스트에 대한 도메인 특화 파인튜닝으로 관련성과 정확도 향상 가능.
  \item \textbf{언어 범위}: 한국어와 영어로 제한. 중국어, 일본어, 독일어 확장으로 5대 화학 산업 문서 언어를 포괄할 수 있음.
  \item \textbf{실시간성}: 특허 인덱스 데이터는 빌드 후 정적. 증분 업데이트 메커니즘과 최신 출원에 대한 실시간 API 통합으로 커버리지 향상 가능.
  \item \textbf{API 의존성}: KOSHA, KIPRIS, KOTRA, MFDS API는 정부 운영 서비스로 가용성, 속도 제한, 사양 변경에 종속. 통합 HTTP 클라이언트의 재시도 로직이 일시적 장애를 완화하나, API 폐지 시 어댑터 업데이트 필요.
\end{enumerate}

\subsection{향후 과제}

\begin{enumerate}
  \item 10명 이상의 화학물질 안전 실무자를 대상으로 한 공식 사용성 연구 수행.
  \item 전체 리빌드 없이 신규 특허 공보를 반영하는 델타 인덱싱 구현.
  \item 화학 안전 코퍼스(MSDS 텍스트, GHS 문서, KOSHA 가이드)에 대한 컴팩트 LLM 파인튜닝.
  \item 파싱된 위험 문구에서 GHS 픽토그램 자동 할당.
  \item 엔터프라이즈 배포를 위한 인증 및 조직별 데이터 격리 추가.
\end{enumerate}
